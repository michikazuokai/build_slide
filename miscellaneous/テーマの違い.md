ざっくり結論から 👇



- **metropolis**：モダンで完成度高い“総合テーマ”。機能豊富・見栄えが揃う反面、設定の流儀に合わせる必要があり、独自改造はややコツが要る。
- **SimpleDarkBlue**：青系のシンプルな（軽量）テーマ。自由度が高く、素直に beamer 標準の仕組みで弄れる一方、細かなコンポーネントは自作・調整が必要な場面が出やすい。

# **違いまとめ**







## **見た目・UI**





- **metropolis**

  

  - 余白設計・タイポグラフィが良く、即戦力で“今っぽい”。
  - セクションページ、プログレスバー、太見出し、最小限の装飾が標準で完備。

  

- **SimpleDarkBlue**

  

  - 青系の配色でスッキリ。フッターやタイトル周りを自前で決めやすい。
  - “必要な分だけ”の装飾で、軽くて速い。

  







## **使い勝手（導入〜運用）**





- **metropolis**

  

  - \usetheme{metropolis} と \metroset{...} で主要オプションが一括管理される（学習コストは低め）。
  - 既定機能が多く「そのまま使う」ほど楽。

  

- **SimpleDarkBlue**

  

  - beamer の素直な API（\setbeamertemplate, \setbeamercolor, \setbeamerfont）でそのまま触れるので、既存の beamer ノウハウが活きる。
  - コンポーネント（セクションページ、進捗表示など）は自作／追設が必要なことがある。

  







## **カスタマイズ性**





- **metropolis**

  

  - 推奨は \metroset とテーマが用意したキー（numbering=none, progressbar=foot など）で調整。
  - 独自の余白・段組・見出しの“深い”改変は、テンプレの上書きや内部マクロの知識が要り、やや玄人向け。

  

- **SimpleDarkBlue**

  

  - beamer 標準 API でほぼ完結。フッターやフレームタイトル、色・フォントを直感的に弄れる。
  - パーツが少ない分、必要機能は自分で積む前提（＝自由度高 / 労力も必要）。

  







## **拡張・相性（日本語・minted 等）**





- **共通**：LuaLaTeX＋luatexja・minted（-shell-escape）はどちらも OK。

- **metropolis**

  

  - 欧文ベースの見出し設計なので、日本語フォントは luatexja-fontspec で適切に設定推奨（太字ウェイトの割当など）。
  - 背景が明るめなので、コードのダーク配色はコントラスト調整が必要な場合あり。

  

- **SimpleDarkBlue**

  

  - 濃い青系を下地にする場合、白字や強調色のコントラストは取りやすい。
  - ただしリンク色（hyperref）やコード枠色は自前で合わせる必要が出がち。

  







## **ビルドの安定性・制約**





- **metropolis**

  

  - 依存がやや多め（フォント・pgf 周り等）。TeX Live 更新直後に微調整が必要になることがある。
  - 既定で hyperref との連携は良好だが、**他所で再度 \usepackage[...]{hyperref} をかけるとオプションクラッシュ**が起きやすい（→ 事前に \PassOptionsToPackage{...}{hyperref} 推奨）。

  

- **SimpleDarkBlue**

  

  - 軽量でエラー原因が追いやすい。
  - 機能を足すほど自前テンプレの一貫性・メンテ工数は増える。

# **実用的な選び分け**





- **速く・整った見た目で量産したい**：metropolis

  （セクションページ・進捗バー・タイポが最初から良い）

- **授業向けにフッターや色、教師用マクロ等を自在に組み込みたい**：SimpleDarkBlue

  （beamer標準 API で柔軟にいじれる）

- **混在運用**なら：

  

  - プレアンブル共通化（共通 .sty に自前マクロ・色・脚注・teacherframe 等）
  - テーマごとに最小差分だけ切替（main_template_org.txt / metro_template_org.txt）

# **すぐ使えるスニペット**







### **metropolis の基本**



```
\usetheme{metropolis}
\metroset{
  sectionpage=progressbar,
  progressbar=foot,
  numbering=fraction
}
% hyperref 衝突回避
\PassOptionsToPackage{unicode,colorlinks,linkcolor=blue,urlcolor=blue}{hyperref}
\usepackage{hyperref}
```



### **SimpleDarkBlue の基本（例）**



```
\usetheme{SimpleDarkBlue}
\setbeamercolor{frametitle}{bg=blue!10, fg=black}
\setbeamercolor{footline}{bg=, fg=gray}
\setbeamertemplate{footline}{
  \leavevmode\hbox to \paperwidth{
    \hspace*{0.3cm}\scriptsize\color{gray}\insertframenumber/\inserttotalframenumber\hfill
  }\vspace{2pt}
}
% hyperref 衝突回避（テーマが読みそうな場合）
\PassOptionsToPackage{unicode,colorlinks,linkcolor=blue,urlcolor=blue}{hyperref}
\usepackage{hyperref}
```

# **デメリット（注意点）だけもう一度**





- **metropolis**

  

  - 内部の美学・余白設計に合わせるのが楽／外れる改変は難しい
  - 依存が多く、TeX Live の更新影響を受けやすい

  

- **SimpleDarkBlue**

  

  - “欲しい機能は自分で足す”前提（セクションページやプログレスなど）
  - 見た目の一貫性は設計次第（ガイド無しで崩し過ぎるリスク）

  





------





## **先生向けのおすすめ運用**





- **共通 .sty（myslides.sty など）**に：

  

  - teacherframe / mypause / 箇条書き・表の標準設定
  - hyperref オプションの前渡し（PassOptionsToPackage）
  - 日本語フォント設定（luatexja-fontspec）

  

- **テーマは授業ごとに切替**し、テンプレは 2 種用意（Simple / Metro）。

- **minted を標準、listings を fallback**（PC室の制限向け）に保持。





どちらも良いテーマです。

「そのまま格好よく」は **metropolis**、

「授業用の細かな都合に合わせたい」は **SimpleDarkBlue**、で使い分けるのが失敗しにくいですよ。
